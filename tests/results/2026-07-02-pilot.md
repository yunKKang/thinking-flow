# 2026-07-02 路由试点（第 1 层）

目的：验证路由 harness 可用性，抽测 3 个场景。运行方式：general-purpose subagent，全新上下文，仅提供五个 description 原文 + 场景消息。失败场景按协议加测至 5 次。

## 结果

| 场景 | 期望 | 运行 | 结果 |
|---|---|---|---|
| F-T（职业纠结+要规划表） | frame | 1 次 | ✅ frame（1/1）。理由正确识别"过早方案驱动"，并预告后续可能 simplify |
| F-C（加类型注解） | 不调用 | 1 次 | ✅ 不调用（1/1）。正确引用 frame 的 "routine execution tasks" 排除条款 |
| L2（中文"奥卡姆剃刀"砍方案） | simplify | 5 次 | ⚠️ simplify 4/5，calibrate 1/5 |

## L2 失败样本判读（人工阅读原文）

第 1 次运行判给 calibrate，理由为"消息焦点是证据变了/涌入新信息后该多相信原结论"——**该理由与消息内容不符，属幻觉式归因**（消息中不存在任何证据更新情节）。后续 4 次全部正确命中 simplify，且理由一致引用 description 中的 Occam's Razor 触发词与"已成形但过繁"条件，输出形状收敛。

判定：跨语言路由基本可用（"奥卡姆剃刀"→"Occam's Razor" 的跨语言匹配在 4/5 情况下成立），但存在约 20% 的偶发误路由，失败模式是把消息强行套进另一个 skill 的触发叙事。

## REFACTOR 候选（暂不执行，待全量路由测试后统一决定）
